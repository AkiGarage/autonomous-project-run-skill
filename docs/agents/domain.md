# Domain docs

This repository uses a single-context layout.

Before exploring domain behavior, read root `CONTEXT.md` and relevant files in `docs/adr/` when they exist. Their absence is not an error; proceed silently until a resolved term or architectural decision justifies creating one.

Use the vocabulary defined by `CONTEXT.md` in issues, tests, and implementation. Surface any conflict with an existing ADR explicitly instead of silently overriding it.
