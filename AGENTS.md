# Repository instructions

- Keep changes small, reviewable, and limited to the public skill.
- Keep `README.md` and `README.ja.md` equivalent when behavior or requirements change.
- Preserve upstream attribution and the MIT notices.
- Run `./scripts/validate.sh` and `./tests/validate-public-surface.sh` before proposing a change.
- Never commit credentials, private paths, internal transcripts, continuity notes, handoff files, or generated archives.
- Do not change repository visibility, publish a release, or bypass branch protections without explicit maintainer approval.

## Agent skills

### Issue tracker

Issues are tracked in this repository's GitHub Issues; external pull requests are not a triage request surface. See `docs/agents/issue-tracker.md`.

### Triage labels
