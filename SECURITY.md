# Security policy

## Supported versions

Before the first stable release, security fixes are applied to the latest commit on `main`. After `0.5.0`, only the latest `0.5.x` release is supported unless a release note states otherwise.

## Reporting a vulnerability

When GitHub's **Report a vulnerability** button is available, use it to submit a private report. Do not include credentials, personal data, private repository content, or exploit details in a public issue.

If private vulnerability reporting is unavailable, open a public issue that asks the maintainer to provide a private contact method. Include no vulnerability details, credentials, proof of concept, or affected private-repository information in that issue.

## Scope

Security reports should concern instructions or bundled resources that could cause unauthorized access, secret exposure, destructive changes, unsafe production actions, or bypasses of repository protections.
